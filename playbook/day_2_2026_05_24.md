## ⚠️ Key Obstacles & Risks Identified

1. **Event Mocking Timing**: In `test_cli.py`, global mock patching on `threading.Event.is_set()` got consumed by standard library synchronization inside `threading.Thread.start()` when starting the background exporter thread.
   * *Solution*: Isolated CLI tests by creating an autouse fixture in `tests/test_cli.py` to mock `start_exporter` and `registry` calls during CLI test execution, preventing thread initialization and port binding conflicts.
2. **Missing Test Fixtures**: Resolved pytest `requests_mock` issues by adding the dependency to `pyproject.toml` and updating the venv.
3. **Mypy Strict Typing**: Fixed type mismatches on dictionary keyword unpacking inside alerting tests and JSON properties in gitops response values.
4. **Prometheus Target Label Overwriting**: In Kubernetes, scraped metrics automatically inherit target metadata labels. Exporter-attached custom `namespace` and `pod` labels get overridden by target fields.
   * *Solution*: Enabled `honorLabels: true` in the ServiceMonitor endpoint to preserve the original scraped namespace, and published workload pod lists under the custom label `pods` while hiding the scraper's `pod` label in Grafana via field overrides.
